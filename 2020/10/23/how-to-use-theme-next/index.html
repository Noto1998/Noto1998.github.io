如何使用NexT主题

发表于 2020-10-23 更新于 2020-10-24

弄好了hexo框架，下一步就是弄外观了，我的好朋友yaolaotou曾说过：

程序员折腾字体、外观，比写代码还起劲，后面不记得了。

一开始我只想自己简单装饰一下，复制一份默认主题看一看代码，里面还在用jQuery，感觉有些年代了。想到很多hexo文章都提到next主题，那就看一看吧，一看确实很简约很戳我，就决定来折腾一下。

我一开始从文章里点开了这个地址，乍看没什么毛病，文章里也是直接clone这个链接，但我多看了眼旁边的官网，发现了一行大字：

The upstream repository of theme NexT has been changed.

原来next主题已经换了新的地址，而原来的readme.md文件并没有更新，看star数可能有一部分人都不知道这回事，比如很多人可能从文章里直接复制clone代码。

一番了解，原来next主题一共有三个仓库，一开始是个人项目（来自厦门的开发者），后来迁移到社群开发，算是一次意外的发现，也给自己一个提醒，要多留意文章的时效性。

咳咳，该说回正题了，目前版本的安装方法是：

npm install hexo-theme-next（要多留意文章时效性哦）

现在不再在\themes\中存放文件夹，而是作为一个包安装在\node_modules\中，不需要任何依赖。之后记得在_config.yml中设置theme: next，不然现在执行hexo命令会报错（我傻傻的以为是安装出错了，其实只是没设置正确而已）。

现在就可以看到应用后的界面了，后续美化可以看next官方文档，next主题提供了很强大的可定制内容，像我目前就设置了自定义页面，还可以搞一搞CDN，图床什么的，暂时还没研究。

另外，推荐用Alternate Theme Config来配置主题而不是直接更改_config.yml文件，简单来说就是创建一个_config.next.yml文件，并复制主题里的_config.yml的内容过来（现在在\node_modules\中），这样不会改动到原来的_config.yml文件。
