在unity里实现跳跃

发表于 2020-11-29

记录下自己踩过的坑，后续可能继续更新。

开始忙毕设，之前早早联系老师定好了用unity引擎制作，但我现在其实更偏爱love2d、pico8这类轻量级引擎了，我也只是做做小游戏，unity有点杀鸡用牛刀了😂，但是已经报上去了也不好麻烦老师改动（毕竟连编程语言都要换），硬着头皮做吧~

第一篇先聊下现在正在实现的跳跃功能，首先了解unity的几种回调。

Update

Update其实就类似love2d或者其他引擎的Update，比较容易理解，画面渲染多少次就执行多少次，所以每秒执行多少次是不固定的，经常搭配currentTime之类的变量使用。

FixUpdate

先说概念，这是一个每秒执行固定次数的方法（比如60次/30次），在GameMakerStudio系列引擎里的update就是这种感觉（可能不叫这个名，很久没用过了）。因为它固定次数，所以你不用关心currentTime，相对来说感觉简单一些。因为时间间隔“固定”，在FixUpdate中适合实现物理运算。

但是用了这么些引擎后，其实在我理解来说，这种FixUpdate其实就是改造过的Update，只不过加入了计数器。比如，在1/60秒内只执行一次，后面的几次忽略掉，可能在最后一个1/60秒会加一个检查，没达到60次就一次性加上。其实之前在love2d上就可以自己实现一个类似的功能。

所以，FixUpdate其实和Update一样，也会受画面渲染次数影响，只是它会自动修正，所以其执行间隔理论上是固定的，但其实不是，所以我在上面打了双引号。

Input Manager

一开始我是看这个比较基础的教程，unity东西很多还是多学习。比如在看教程时就了解到unity的Input Manager，这是一个类似映射表的东西，可以自己添加按键，然后设定对应的键盘/手柄按键。

之前在love2d就自己造过类似的映射轮子，当时还自己实现了isPressed检测（即unity中的GetButtonDown，其实引擎不自带这种功能没什么道理……），现在可以直接用就很方便。

而且在Input类的方法中，GetAxisRaw方法可以把两个按键之间的按键关系映射成1、-1、0，这个功能很简单但也很方便，可以一行就完成添加移动加速度/左右转向。我记得初学GMS编程时就是学习如何实现这种方法，真是感叹~

顺带一提，Unity有一套新的Input System，还在不断开发中，看起来是可以建立多套映射的样子，以后可以试试。
